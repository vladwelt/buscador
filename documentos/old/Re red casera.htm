DHCP es un servidor para asignar direcciones IP dinamicas  a una red..
es decir que cada vez que uno de los clientes windows se conecte a la
red recibira una ip diferente..  ESTO NO ES NECESARIO para la red que
piensas montar.. en una red tan pequeña basta con utilizar direcciones
ip clase C privadas .. 192.168.

SAMBA es un servidor para compartir recursos con el protocolo que
utiliza windows para el entorno de red.. este PUEDE ser nesesario o no
..dependiendo si deseas que desde los clientes windows se puedan abrir
directorios que estan en el servidor linux o si linux tiene conectada
una impresora que utilizaran los clientes windows..(al contrario tambien
funciona).  Para esto buesca en es.tldp.org el manual de samba. 

IPTABLES: es INDISPENSABLE .. TIENES QUE COLOCARLO.. es el sistema de
firewall para protejer tu red interna.. y ademas sirve para realizar el
forward (reenvio de paquetes) para que los clientes windows puedan
conectar a internet.

Las 3 cosas son independientes.. te sugiero que primero configures tu
red con direcciones ip fijas .. algo asi ..

Linux 192.168.1.1
Windows_1 192.168.1.2
windows_2 192.168.1.3

Mascara 255.255.255.0

por supuesto linux debe tener otra interfaz (modem eth1) que permita la
conexion a internet..

Luego has el reenvio de paquetes..para eso utilizas iptables.. y luego
pones el firewall tambien con iptables..

Despues de que esto funcione te tomas un par de dias (3 o 4) de
vacaciones para relajar tu mente.. :-D

Ahora si atacas el samba.. y despues "al infinito y mas alla" como decia
Buzz ;-)
